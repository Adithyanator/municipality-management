# Project Roadmap

## Purpose

This roadmap defines the documentation-first project progression for the AI-Powered Smart Municipal Citizen Assistance and Staff Attendance Management System.

## Phase 1 - Repository and Architecture Foundation

Status: Complete

Deliverables:

- Git repository initialized.
- Root project structure created.
- Approved architecture documents organized.
- Team ownership structure documented.
- Branch strategy created and pushed.

## Phase 2 - Documentation Organization

Status: In Progress

Deliverables:

- Numbered documentation folders.
- Folder-level README files.
- Architecture review checklist.
- Team assignments document.
- Project management and handover documentation structure.

## Phase 3 - API and Database Planning

Status: Not Started

Deliverables:

- Backend API module inventory.
- MongoDB schema planning documents.
- Index and validation plan.
- Authentication, RBAC, audit, and report export contract notes.

## Phase 4 - UI and Frontend Planning

Status: Not Started

Deliverables:

- React route map.
- Portal-specific screen inventory.
- Component and layout inventory.
- Frontend access-control and route-guard notes.

## Phase 5 - AI and Face Recognition Planning

Status: Not Started

Deliverables:

- Chatbot RAG workflow notes.
- Prompt and guardrail inventory.
- Face enrollment, liveness, matching, and attendance workflow notes.
- Human review and fallback workflow notes.

## Phase 6 - Implementation

Status: Not Started

Deliverables:

- Frontend implementation.
- Backend implementation.
- AI services implementation.
- Database migrations, indexes, seeders, and validation rules.

## Phase 7 - Verification and Handover

Status: Not Started

Deliverables:

- Architecture review completion.
- Security review completion.
- Test evidence and known limitation notes.
- Handover documents.
